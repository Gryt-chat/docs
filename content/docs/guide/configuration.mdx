## Web Client

The web client works out-of-the-box. Optional settings in `packages/client/.env`:

```bash
VITE_AUDIO_SAMPLE_RATE=48000
VITE_AUDIO_BUFFER_SIZE=256
```

Audio settings (volume, noise gate, device selection) are configured through the UI at runtime.

## Signaling Server

Create a `.env` file in `packages/server/` based on `example.env`:

```bash
PORT=5000

# SFU websocket (internal, container-to-container)
SFU_WS_HOST="ws://sfu:5005"

# SFU websocket that browsers connect to (public URL).
# Comma-separated for multi-network (client auto-selects fastest).
SFU_PUBLIC_HOST="wss://sfu.example.com"

STUN_SERVERS="stun:stun.l.google.com:19302"

SERVER_NAME="My Brand New Server"
SERVER_INVITE_ONLY="false"
CORS_ORIGIN="https://app.gryt.chat"
SERVER_PASSWORD="123"

# Password brute-force protection (optional)
# Two-tier: per (IP + user) and per IP. Cooldowns escalate exponentially.
# SERVER_PASSWORD_MAX_RETRIES=5            # attempts before lockout
# SERVER_PASSWORD_RETRY_WINDOW_MS=300000   # sliding window (5 min)
# SERVER_PASSWORD_RETRY_COOLDOWN_MS=60000  # base cooldown (1 min)
# SERVER_PASSWORD_MAX_COOLDOWN_MS=3600000  # escalation cap (1 hour)
# SERVER_PASSWORD_IP_MAX_RETRIES=15        # per-IP attempts before lockout

# Authentication
GRYT_AUTH_API=https://auth.gryt.chat

# ScyllaDB
SCYLLA_CONTACT_POINTS=127.0.0.1
SCYLLA_LOCAL_DATACENTER=datacenter1
SCYLLA_KEYSPACE=gryt

# S3 / Object storage
S3_REGION=auto
S3_ACCESS_KEY_ID=
S3_SECRET_ACCESS_KEY=
S3_BUCKET=gryt-bucket
S3_FORCE_PATH_STYLE=false

NODE_ENV=development
DEBUG=gryt:*
```

### MinIO (self-hosted S3)

```bash
S3_ENDPOINT=http://localhost:9000
S3_ACCESS_KEY_ID=admin
S3_SECRET_ACCESS_KEY=change-me-please
S3_BUCKET=gryt-bucket
S3_FORCE_PATH_STYLE=true
```

```bash
docker run -d --name minio \
  -p 9000:9000 -p 9001:9001 \
  -v /srv/minio/data:/data \
  -e MINIO_ROOT_USER=admin \
  -e MINIO_ROOT_PASSWORD=change-me-please \
  quay.io/minio/minio server /data --console-address ":9001"

docker run --rm --network host \
  -e MC_HOST_minio=http://admin:change-me-please@localhost:9000 \
  quay.io/minio/mc mb --ignore-existing minio/gryt-bucket
```

## SFU Server

Create a `.env` file in `packages/sfu/`:

```bash
PORT=5005
STUN_SERVERS=stun:stun.l.google.com:19302,stun:stun1.l.google.com:19302
ICE_UDP_PORT_MIN=10000
ICE_UDP_PORT_MAX=10019
LOG_LEVEL=info
MAX_CONNECTIONS=1000
```

For production, also consider setting `ICE_ADVERTISE_IP` if the SFU host is behind NAT.
For multi-network setups (e.g. LAN party + public), both variables accept comma-separated values:

```bash
ICE_ADVERTISE_IP=203.0.113.10,192.168.1.100
```

## Authentication

Authentication is centrally hosted at `https://auth.gryt.chat`. No additional configuration is needed by default.

To require Gryt accounts on your server:

```bash
GRYT_AUTH_MODE=required
GRYT_OIDC_ISSUER=https://auth.gryt.chat/realms/gryt
GRYT_OIDC_AUDIENCE=gryt-web
```

To disable authentication:

```bash
GRYT_AUTH_MODE=disabled
```

See the [Server docs](/docs/server) for more on auth integration.

## STUN + SFU UDP ports (production)

This project does **not** require a TURN server. Instead:

1. Configure STUN servers (`STUN_SERVERS`)
2. Pin a dedicated SFU UDP port range (`ICE_UDP_PORT_MIN` / `ICE_UDP_PORT_MAX`)
3. Open that UDP range on the SFU host firewall

```bash
STUN_SERVERS=stun:stun.l.google.com:19302,stun:stun1.l.google.com:19302
ICE_UDP_PORT_MIN=10000
ICE_UDP_PORT_MAX=10019
```

## Access control with CORS

The `CORS_ORIGIN` environment variable controls which web origins can talk to your server.
This acts as a simple access gate: only clients loaded from a listed origin will be able to connect.

```bash
# Only allow the hosted client and your custom domain
CORS_ORIGIN="https://app.gryt.chat,https://gryt.example.com"
```

Because the Electron desktop app defaults to a specific local port (e.g. `http://localhost:3666`),
you can separate dev and production environments by listing different ports:

```bash
# Production server — only accepts the production client
CORS_ORIGIN="https://app.gryt.chat"

# Dev server — only accepts the local dev client
CORS_ORIGIN="http://localhost:3777"
```

A dev client on `localhost:3777` cannot connect to a production server that only allows
`https://app.gryt.chat`, and vice versa. This gives you environment isolation without
any extra infrastructure.

<Callout type="info">
CORS is enforced by the browser and by the server's Socket.IO and REST middleware.
It is not a security boundary against non-browser clients — someone could still connect
with `curl` or a modified client. For real authentication, enable
[Gryt Auth](/docs/guide/configuration#authentication) or set a `SERVER_PASSWORD`.
</Callout>

## Security best practices

- Never commit `.env` files to version control
- Use strong, unique passwords for `SERVER_PASSWORD`
- Use HTTPS/WSS in production
- Configure CORS origins to match your domain
- Run containers as non-root users
